1. Initial testing with VD100_10G_ether_dual
--------------------------------------------

For this the VD100_10G_ether_dual design was connected to a T1700G-28TQ switch:
a. VD100 port 0 connected to switch port 27 via DELL PKR0R 10GB RJ-45 SR 30m Transceivers
b. VD100 port 1 connected to switch port 28 via 10Gtek AXS85-192-M3 is SFP+ 10Gb/s, 10GBase-SR, MMF, 850nm 300M Transceivers,
   with a multi-mode fibre.
c. Only one VD100 port connected to the switch at once.
d. 10G DAC cable between switch ports 25 and 26.
e. Switch port VLAN configuration:
   - Port 25 untagged member of VLAN 1025
   - Port 26 untagged member of VLAN 1026
   - Ports 27 and 28 tagged members of VLAN 1025-1026

2. Add logic to wait for link to be ready
-----------------------------------------

Noticed that a VFIO open or close, which issues a PCIe Hot Reset, caused the link LEDs on the switch to show the links
go down for about one second and then back up.

Changed mrmac_switch_test to check for the latched receive status indicating the link had gone down to wait for the link
to be ready to receive and an additional delay for the switch to be ready (as found with dpdk_switch_test).

3. Allow number of queued transmit buffers to be increased when not rate limited
--------------------------------------------------------------------------------

When testing two 10G ports the achieved bandwidth to the injection switch was 8.4Gbps.

The maximum number of pending transmit frames was limited to the number of ports tested, i.e. 2, regardless of if the
transmit needed to be rate limited to protect loss in the switch under test.

Adjusted so that:
a. When rate limited, as before the maximum number of pending transmit frames is the number of tested ports.
a. When not rate limited the maximum number of pending transmit frames is increased to 48 (NUM_DEFINED_PORTS)
